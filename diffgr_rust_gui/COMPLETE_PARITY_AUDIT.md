# Complete Python Parity Audit

Scope: the original DiffGR Python package in `compat/python`, especially the 31
historical command scripts under `compat/python/scripts`.

## Result

All 31 script names have:

- a native Rust CLI alias in `diffgrctl`,
- a PowerShell wrapper under `scripts/*.ps1`,
- a shell wrapper under `scripts/*.sh`,
- an exact vendored Python compatibility path under `compat/python/scripts/*.py`,
- an entry in `PYTHON_PARITY_MANIFEST.json` with argparse option metadata.

The native Rust implementation is intended for normal use.  The compatibility
path is intended for byte/behavior parity when validating against old Python
outputs or old Python unit-test expectations.

## Native Rust parity improvements in this package

Compared with the previous archive, this package adds or tightens:

- state overlay support for the `view-diffgr` CLI path,
- Python-compatible `diff_diffgr_state` human output by default,
- rejection of mixed explicit tokens with impact-plan state apply,
- Python-compatible `merge_diffgr_state` requirement that `--output` is present unless `--preview`,
- `split_group_reviews --manifest` handling,
- coverage command text output and non-success result when coverage is invalid,
- `impact_report --grouping` / `--similarity-threshold` option acceptance on the native path,
- `PYTHON_PARITY_MANIFEST.json` generated from the vendored Python argparse definitions,
- `-CompatPython` / `--compat-python` exact fallback for every historical script wrapper.

## Source-level strict parity added

This package includes `COMPLETE_PYTHON_SOURCE_AUDIT.json`, which records hashes
for every non-cache file copied from the original Python app.  The verifier below
checks source hashes, wrapper coverage, Python compilation, and smoke commands.

```powershell
.\windows\python-compat-verify-windows.ps1 -Json
```

```bash
./compat-python-verify.sh --json
```

The expected result is:

```json
{
  "ok": true,
  "sourceFilesVerified": 163,
  "pythonScriptsVerified": 31,
  "excludedCacheFiles": 99,
  "compiled": true,
  "smoke": true,
  "pytest": false
}
```

The 99 excluded files are only `__pycache__/*.pyc`; they are regenerated by
Python and are not part of source functionality.
